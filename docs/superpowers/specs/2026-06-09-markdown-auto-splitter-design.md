# Markdown 自动分割器设计文档

## 概述

MarkdownAutoSplitter 是一个工具类，用于将解析后的 Markdown 内容自动分割成适合向量化并存入向量数据库的数据块。它能智能地处理 Markdown 结构元素（标题、表格），生成有意义的文本片段。

## 需求

- **输入**：已解析的 Markdown 文本字符串
- **输出**：Spring AI `Document` 对象列表
- **关键规则**：
  - 表格作为完整单元保留（不分割）
  - 标题引导分割边界
  - 数据块大小由字符数控制

## 架构

```
┌─────────────────────────────────────────────────────┐
│                  MarkdownAutoSplitter                │
├─────────────────────────────────────────────────────┤
│  - maxChunkSize: int = 1000                        │
│  - minChunkSize: int = 100                         │
│  - includeTitleInChunk: boolean = false             │
└─────────────────────────────────────────────────────┘
         │
         ▼
┌─────────────────────────────────────────────────────┐
│              List<Document> split()                 │
├─────────────────────────────────────────────────────┤
│  1. 识别 Markdown 结构（标题、表格）                 │
│  2. 构建段落单元                                    │
│  3. 应用大小约束                                    │
│  4. 返回 Document 列表                              │
└─────────────────────────────────────────────────────┘
```

## 核心算法

### 第 1 步：预处理
- 如果存在 YAML 前置元数据，则移除

### 第 2 步：结构识别
- 通过正则表达式匹配 `#` 标题（h1-h6）
- 通过正则表达式匹配表格边界（`|---|`）
- 记录元素类型和位置

### 第 3 步：段落单元构建
| 场景 | 处理方式 |
|------|----------|
| `# 标题` | 创建新的段落单元 |
| `## 子标题` | 内容合并到当前段落 |
| 表格 | 整个表格作为一个单元 |
| 普通文本 | 追加到当前段落 |

### 第 4 步：基于大小的分割
- 遍历段落单元
- 累计字符数
- 如果超过 maxChunkSize(1000)：写入新数据块
- 如果剩余 < minChunkSize(100)：合并到前一个数据块

### 第 5 步：生成 Document 对象
- 设置内容 + 元数据

## 类设计

```java
public class MarkdownAutoSplitter {
    
    private final int maxChunkSize;      // 默认：1000
    private final int minChunkSize;       // 默认：100
    private final boolean includeTitleInChunk; // 默认：false
    
    /**
     * 将 Markdown 分割成数据块
     * @param markdown 原始 Markdown 文本
     * @return Document 对象列表
     */
    public List<Document> split(String markdown);
}
```

## 处理规则

| 场景 | 处理方式 |
|------|----------|
| `# 标题` | 开始新数据块，标题作为元数据 |
| `## 子标题` | 内容合并到当前数据块 |
| 表格 | 整个表格保留，不分割 |
| 超大数据块 | 在边界处截断 |
| 过小数据块 | 与前一个数据块合并 |
| 嵌套表格 | 不支持（按 Markdown 规范），视为普通文本 |
| 特殊字符 | 正常处理，不在字符中间断开 |
| 大文档 | 增量处理，内存安全 |

## 错误处理

- 空输入 → 返回空列表
- 解析失败 → 抛出 IllegalArgumentException
- 异常不在内部捕获（由调用方决定处理方式）
